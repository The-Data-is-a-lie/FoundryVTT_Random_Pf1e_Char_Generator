Version 2.0.0 (2026-07-04)

- Changed: The Biography tab now shows the backend's coherent prose backstory (new `backstory` payload field) instead of the raw labeled field-by-field dump; that raw dump (family, mannerisms, personality, flaws, appearance, region, professions, schools, etc.) moved to the Notes tab. If no backstory is present (e.g. an un-redeployed backend), it falls back to the old behavior (raw dump in Biography). New backstoryToHtml() helper escapes the text and renders paragraph breaks.
- Fixed: Path of War attack-buff conditionals (with a [[ ]] rider in the name) no longer crash the attack roll ("SyntaxError: Expected ] but [ found"). pf1 embeds the conditional NAME as the attack-roll term's flavor — (formula)[name] — and the [[ ]] nested brackets broke the d20 parser (damage conditionals were unaffected). The generator now gives attack-target conditional modifiers their own [maneuver name] flavor, so pf1 uses the formula as-is and never embeds the name — keeping the clickable [[ ]] rider intact.
- Changed: The main weapon now also gets the size-scaling setup — a 2nd "Don't Touch" action (a copy of the attack, read by the script as the pristine base damage) plus the "Scaling Weapon Damage" script call — so damage scaling works when rolling the weapon directly in the Combat tab (previously only the separate attack item had it). The separate attack item is still created.
- Fixed: The sizefordamage feature now sits at the BOTTOM of the Class Features section instead of the top. It had sort 0 while other class features have very large sorts, and pf1 orders sections ascending; it now gets a sort just above the largest class feature.
- Changed: Path of War maneuver conditionals (on the weapon's attack) now also carry a descriptive RIDER in the conditional name — the saving throw, ability damage/drain, inflicted condition, or other key non-damage effect — with the numbers in [[ ]] inline rolls (e.g. "(Strike) Heavenly Blade of the Scarlet Throne: Will Save [[ 19 + @abilities.int.mod ]] or paralyzed [[1d4]] rounds"). The save DC uses 10 + maneuver level + the character's REAL initiation modifier, resolved by new helper maneuverInitAttr() (initiator class -> its class initiatorAttr; Martial Training -> highest mental), so it matches pf1-pow's own maneuver DC. Save/condition/ability-damage-only maneuvers (no numeric modifier) now produce a conditional too (previously skipped). templates/character_sheet_folder/maneuver_changes.json was regenerated for all 677 strikes/boosts (numeric modifiers + riders); @INITMOD is the per-character substitution token. Numeric damage/attack stays in modifiers as before.
- Fixed: Generated weapons now appear in the Combat (Attacks) tab, not only in Inventory/Quick Actions. pf1 v11 lists a weapon in that tab only when system.showInCombat is true (it does NOT check equipped), and generated weapons left it falsy. processItem() now sets system.showInCombat = true on weapons (only weapons — armor/wondrous items are unaffected).
- Added: Size-based damage scaling on every generated sheet. A new "sizefordamage" feature exposes the @resources.sizefordamage charge (default 0), and each generated weapon now also produces a separate ATTACK item (pf1 "Create Attack" equivalent) carrying a "Scaling Weapon Damage" script call (category Use) plus two actions in order — "Attack" (the rollable copy, inheriting any maneuver conditionals) and "Don't Touch" (a duplicate the script reads as the pristine base damage to scale from by size). Raising sizefordamage scales the maneuver/conditional dice up at roll time (needs the ckl-roll-bonuses module). The original weapon item is left untouched and stays in Combat alongside the attack. New templates sizefordamage_feature.json + scaling_weapon_damage.json; new addSizeForDamageFeature()/createScalingAttackItem() in modify-abilities.js.
- Added: Path of War maneuvers now add conditional modifiers to the generated character's main weapon. Each known strike/boost/counter that has a clean combat effect (curated in templates/character_sheet_folder/maneuver_changes.json) becomes a default-OFF conditional on the weapon's attack action, named "(Strike)/(Boost)/(Counter) <name>", that you tick in the attack dialog when you use the maneuver (e.g. "(Strike) Sting of the Rattler" adds +1d4 damage). Real per-hit dice are used (buff changes can't roll dice). Maneuvers without a clean numeric effect, and stance dice damage, stay description-only.
- Fixed: Item sheets (weapons, and potentially feats/classes/races/spells) no longer crash with "a.standard is not iterable". pf1 v11 requires item group fields (weaponGroups, weaponProf, armorProf, languages, descriptors, subschool, creatureTypes, creatureSubtypes) to be stored as BARE ARRAYS of keys — its item prepareData only builds the iterable { standard, custom } model from an array. The harvested templates carried legacy pf1 v10 object shapes ({base:[...]} / {value,custom}), which v11 leaves untouched, so .standard was undefined when the sheet rendered. The generator now normalizes every exported item's group fields to bare arrays. (This supersedes/reverts an earlier change that wrongly wrapped weaponGroups into { base: [...] } — that was the broken direction.)
- Fixed: Generated max-level (level 20) characters no longer show Level 0 / "Your character is missing a class" and no longer require a manual "Level Up" click. The class item is injected at the every_class.json template level (20), so adjustLevel's update to the same level 20 was a no-op that never triggered pf1's character-level recompute; it now nudges the class level before setting the target so pf1 always recomputes. (Sub-20 characters were unaffected because their level already differed from the template.)
- Added: scripts/pow-sort-override.js also re-registers pf1-pow's filteredLevelsArray helper so the Path of War tab shows a level section for every maneuver level actually on the actor. Martial Training characters now receive maneuvers up to their feat-tier level (e.g. a depth-4 chain grants level-4 maneuvers), which pf1-pow's archetype max-maneuver-level formula would otherwise hide at some class levels. Safe for normal actors (their maneuvers never exceed their own max, so the output is unchanged).
- Changed: Path of War maneuvers/stances are now created as native pf1-pow.maneuver items instead of feat items, so they appear in pf1-pow's own "Path of War" actor tab grouped under the class. Items are cloned from the pf1-pow.disciplines compendium when the name matches (clean descriptions and icons; apostrophe-insensitive matching) and synthesized from the backend's maneuvers_desc_dict otherwise. Names are prefixed (Strike)/(Boost)/(Counter)/(Stance); readied maneuvers start ready with a charge. The old feat-item section remains as an automatic fallback when the pf1-pow module is disabled.
- Added: Martial Training characters (non-initiators with maneuvers) get system.maneuverProgression = {classType "archetype", type "regular", initiatorAttr} on their class item plus the pf1-pow maneuverAttr actor flag, using the backend's new initiation_stat export (highest final Int/Wis/Cha; client-side fallback for un-redeployed backends). Path of War initiator classes are untouched - their every_class.json items already carry maneuverProgression.
- Added: Each chosen stance also becomes an inactive temporary buff under a "____ Path of War ____" divider in the Buffs tab (new space_Path_of_War_buffs.json). Mechanical changes/context notes come from the curated stance_changes.json (22 stances seeded; initiator-level scaling uses @pow.initLevel with ifelse()/gte() - no JS ternaries); stances without an entry are description-only toggles.
- Added: scripts/pow-sort-override.js re-registers pf1-pow's sortManeuvers Handlebars helper on ready, ordering each level section of the Path of War tab by discipline then Strike/Boost/Counter/Stance then level then name (pf1-pow's own helper interleaves types). Note: the tab's per-level section headers are rendered around the helper by pf1-pow.hbs, so level remains the outer grouping.
- Changed: The "Skill Ranks Based on Wisdom/Charisma" custom-buff pick now prefers the backend's initiation_stat export (final scores including inherents and level-up bumps) over comparing raw ability scores; raw-score comparison remains the fallback.
- Fixed: every_class_MODS.json was missing all Path of War classes, so generating a PoW initiator (Harbinger/Medic/Mystic/Warder/Warlord) on the modded sheet produced an actor with no class item and orphaned maneuvers (initiator level 0, no Path of War tab). The 5 PoW class blocks (class item + ability items + maneuverProgression) are now backfilled from every_class.json via the new idempotent tools/backfill_pow_classes_into_mods.js (run with: node tools/backfill_pow_classes_into_mods.js). Stalker and Zealot remain pending in both bundles until the pf1-pow compendium ships them.
- Added: Class bonus feats now display as "(Class) (level): Feat" (e.g. "Fighter 1: Weapon Focus"), using the backend's class_feat_labels field.
- Added: Teamwork feats (Hunter/Inquisitor, Cavalier/Samurai) are now labeled the same way (e.g. "Inquisitor 3: Coordinated Defense") instead of the generic "Class Bonus Feat".
- Added: Optional "Add custom buffs" toggle - when enabled, injects a curated buff set (5 combat buffs, Professions, Skill Synergies, a Wisdom/Charisma skill-ranks buff chosen by highest mental ability, and an Acrobatics Speed buff) into the generated character.
- Fixed: Skill ranks now populate on generated character sheets (they were silently dropped because the backend's skill_ranks string was never parsed).
- Fixed: Craft, Perform, and Profession ranks are now written as pf1e subskills (crf/prf/pro -> subSkills) instead of the ignored container rank.
- Added: Craft now displays its specialization as "Craft: <type>" (e.g. "Craft: Pottery"), from the backend's craft_type field (falls back to "Craft" if the backend hasn't been redeployed).
- Changed: Every chosen profession now becomes its own "Profession: <name>" subskill with ranks split evenly across them, instead of only showing the first profession.
- Added: Bloodline bonus feats (Sorcerer/Bloodrager) now appear on generated sheets, labeled by granting class and level (e.g. "Sorcerer 7: Combat Casting"), from the backend's bloodline_feats/bloodline_feat_labels fields.
- Fixed: Unchained classes (Barbarian/Monk/Rogue/Summoner Unchained) now create the correct "(Unchained)" class item on the sheet instead of the base class. The module now uses the backend's new c_class_display field (already in "Barbarian (Unchained)" form) for the class-item lookup and level-setting, since capitalizeWords couldn't reconstruct the capital "U". Falls back to the base class name if the backend hasn't been redeployed.
- Fixed: Generated characters no longer pick up a spurious second class at level 20 (e.g. Monk (Unchained) also getting a Ninja 20, Rogue (Unchained) a Samurai 20, Summoner (Unchained) a Vigilante 20, Investigator a Shifter 20). The class_list that collectItems() uses as ability boundaries was missing Ninja, Samurai, Shifter, and Vigilante, so the class listed just before each of them in every_class.json bled into it. All 44 classes are now in class_list.
